# Multi-Index Batch Retrieval<a name="ZH-CN_TOPIC_0000001456535132"></a>

When the `distances` values are the same, multi-Index batch retrieval uses a different TopK ranking algorithm than single-Index retrieval. Therefore, the labels in the returned TopK results can differ.

## `Search` (`AscendIndex`)<a name="ZH-CN_TOPIC_0000001456854904"></a>

<a id="table7235918388"></a>

| API Definition | ```void Search(std::vector<AscendIndex *> indexes, idx_t n, const float *x, idx_t k,float *distances, idx_t *labels, bool merged);``` |
| --- | --- |
| Description | Provides an interface for searching feature vectors across multiple `AscendIndex` instances. It returns the distances and IDs of the `k` most similar features based on the input feature vectors.<br>The following algorithms are currently supported:<br>Subtypes derived from `Index`: `AscendIndexSQ` (`QuantizerType` = `QT_8bit`), `AscendIndexFlat` (`FlatIP`, `FlatL2`), and `AscendIndexIVFSP`. |
| Input | `std::vector<AscendIndex *> indexes`: Multiple indexes to search.<br>`idx_t n`: Number of queries to search.<br>`const float *x`: Query feature vectors to search.<br>`idx_t k`: Number of most similar results to return.<br>`bool merged`: Whether to merge the results from search across multiple `Index` instances. |
| Output | `float *distances`: Distance values between the query vectors and the `k` nearest vectors.<br>`idx_t *labels`: IDs of the `k` nearest vectors to the queries. |
| Returns | None |
| Constraints | The `Index` instances participating in search must be created on the same card. The supported `indexes` types are as follows. `indexes` may point to `AscendIndexSQ`, and `QuantizerType` must be `QT_8bit`, with 0 < `indexes.size()` ≤ 10000. `indexes` may point to `AscendIndexIVFSP`, and the corresponding `QuantizerType` must be `QT_8bit` and `MetricType` must be `METRIC_L2`, with 0 < `indexes.size()` ≤ 10000. The `AscendIndexIVFSP` indexes participating in search must share the same codebook in shared memory. You can create instances with the shared-codebook constructor provided by `AscendIndexIVFSP` or through the `loadAllData` interface. `indexes` may point to `AscendIndexFlat`, with 0 < `indexes.size()` ≤ 10000. `n` must not exceed 1024. `k` must not exceed 1024. `x` must be a non-null pointer, and its length must be `dim * n`, otherwise out-of-bounds reads or writes may occur and cause the program to crash. `distances` and `labels` must be non-null pointers. When `merged = true`, their length must be `k * n`, otherwise out-of-bounds reads or writes may occur and cause the program to crash. When `merged = false`, their length must be `indexes.size() * k * n`, otherwise out-of-bounds reads or writes may occur and cause the program to crash. |

The `AscendIndexIVFSP` indexes participating in search must share the same codebook in shared memory. You can create instances with the shared-codebook constructor provided by `AscendIndexIVFSP` or through the `loadAllData` interface. `n` must not exceed 1024. `k` must not exceed 1024. `x` must be a non-null pointer, and its length must be `dim * n`, otherwise out-of-bounds reads or writes may occur and cause the program to crash. `distances` and `labels` must be non-null pointers.
