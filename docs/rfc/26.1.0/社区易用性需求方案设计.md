# 1. 概述

## 1.1 简介

本提案旨在全面提升 IndexSDK 的获取与部署易用性，通过三项核心改进：

- **命令行下载**：提供 `wget` 直链下载，替代手动网页下载+上传的繁琐流程。
- **容器镜像支持**：提供官方 Dockerfile 及预构建镜像（支持 310P/A2/A3，x86/arm，openEuler24.03/Ubuntu22.04），实现开箱即用的容器化环境。
- **免登录下载 + 许可协议命令式确认**：优化昇腾社区下载体验（无需登录），并在安装时支持静默接受协议，适配自动化场景。

## 1.2 动机

当前 IndexSDK 的交付和使用存在以下痛点：

| 痛点 | 具体表现 |
| --- | --- |
| **无命令行下载** | 用户必须手动从昇腾社区下载 `.run` 包，无法集成到自动化脚本或 Dockerfile 中。 |
| **镜像覆盖不全** | 无官方 Docker 镜像，用户自行容器化时需摸索基础镜像、依赖、环境变量，耗时且易错。 |
| **下载需登录** | 昇腾社区要求登录后才能下载软件包，不方便客户使用。 |

提升易用性后，用户可通过一行命令下载、一条指令构建/拉取镜像、静默安装，显著降低使用门槛。

## 1.3 目标

### 目标

1. **wget 命令行下载**
   - 提供稳定的直链 URL（基于 GitCode Release）。
   - 支持 x86_64 和 aarch64 两种架构。

2. **Docker 镜像支持**
   - 提供官方 Dockerfile（多阶段构建）。
   - 预构建镜像发布至 AscendHub。
   - 覆盖硬件：Atlas 310P、910、A3、950。
   - 覆盖架构：x86_64、aarch64。
   - 覆盖操作系统：openEuler 24.03、Ubuntu 22.04。
   - 镜像内部预装 IndexSDK 并配置好环境，用户可直接运行用例。

3. **免登录下载 + 协议静默确认**
   - 昇腾社区支持无需登录即可下载 IndexSDK 软件包（通过勾选协议代替登录验证）。
   - IndexSDK 的 `.run` 安装包支持 `-quiet` 参数，实现命令式确认，避免交互阻塞。

### 非目标

- 不改变 IndexSDK 核心功能与 API。
- 不提供 Windows/macOS 镜像。
- 不负责社区前端登录流程的改造（仅提需求，由社区平台团队实施）。
- 不提供 apt/yum 仓库。

# 2. 用例分析

| 用例场景 | 功能要求 | 关键指标 |
| --- | --- | --- |
| **自动化下载与安装** | `wget <url> && ./xxx.run --quiet --install` 无交互完成 | 下载成功率 ≥ 99%，安装耗时 < 1 分钟 |
| **容器化开发/测试** | `docker pull` 后直接运行，无需手动配置环境 | 启动后即可编译运行入门用例 |
| **多硬件适配验证** | 同一镜像可在 310P / 910 / A3 / 950 上运行用例 | 用例通过率 100% |
| **离线环境部署** | 用户在有网环境下载镜像包并导出，再导入离线服务器 | 镜像导出/导入功能正常 |
| **协议合规** | 下载时用户必须勾选协议；安装时支持静默接受 | 法务审核通过 |

# 3. 方案设计

## 3.1 总体方案

采用**三轨并行**的交付方式：
[用户] ──┬─ wget 直链 ──→ 手动安装
         ├─ docker pull ──→ 容器运行
         └─ 社区免登录下载 + 静默安装 ──→ 自动化脚本

### 3.1.1 wget 命令行下载

- 发布至 GitCode Release：`https://gitcode.com/Ascend/IndexSDK/releases/`

### 3.1.2 Docker 镜像制作

**基础镜像选择**：

| 操作系统 | 架构 | 基础镜像 | 理由 |
| --- | --- | --- | --- |
| openEuler 24.03 | x86_64 / aarch64 | `openeuler/openeuler:24.03` | 与昇腾原生环境一致，驱动兼容性好 |
| Ubuntu 22.04 | x86_64 / aarch64 | `ubuntu:22.04` | 社区流行，用户熟悉 |

多架构构建：使用 docker buildx 同时构建 linux/amd64 和 linux/arm64。

镜像发布：推送至华为云 SWR（swr.cn-south-1.myhuaweicloud.com/ascendhub/indexsdk）。

验证：在 310P、910、A3、950 硬件上分别运行入们用例。

### 3.1.3 免登录下载与协议命令式确认

免登录下载：

当前昇腾社区要求用户登录后才能下载。需与社区平台团队沟通，改为：下载时弹出协议勾选框，用户勾选后直接下载，无需登录。

此改动适用于所有 IndexSDK 版本的下载页面。

命令式协议确认：

修改 IndexSDK 的 .run 安装脚本（由 makeself 生成）。

若用户执行 ./xxx.run -quiet，则跳过交互式协议确认，直接进入安装流程。

示例：

bash
交互式（默认）
./Ascend-mindxsdk-mxindex_26.0.0_linux_x86_64.run

静默安装
./Ascend-mindxsdk-mxindex_26.0.0_linux_x86_64.run --install -quiet

# 4. 安全隐私与 DFX 设计

| 维度 | 设计要点 |
| --- | --- |
| **协议合规** | 免登录方案通过勾选协议满足法务要求；静默安装仅适用于已通过其他方式确认的用户 |
| **兼容性** | Docker 镜像内 IndexSDK 版本与宿主机驱动版本解耦，但需确保 NPU 设备映射（`--device=/dev/davinciX`） |
| **可测试性** | 提供 CI 流水线：每次 IndexSDK 发版自动构建多架构/多OS镜像，并在真实硬件上运行冒烟测试 |
| **可维护性** | Dockerfile 使用多阶段构建减小体积；使用 `ARG` 支持版本参数化 |

# 5. 缺点与风险

| 风险 | 影响 | 应对措施 |
| --- | --- | --- |
| **免登录下载可能被滥用** | 增加带宽成本，但无安全风险 | 可接受；若流量过大再考虑限速或验证码 |
| **Docker 镜像体积过大** | 拉取慢 | 采用多阶段构建、清理缓存 |
| **不同硬件（310P/910/A3/950）驱动差异** | 同一镜像可能在某型号上失败 | 镜像内不包含驱动，依赖宿主机驱动；文档明确要求 |
| **openEuler 24.03 基础镜像尚未稳定** | 可能出现依赖缺失 | 与 openEuler 社区保持同步；同时提供 Ubuntu 镜像作为备选 |
